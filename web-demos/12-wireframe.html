The System

Three modules.
One defensive layer.

Strictly protective by construction — Genome Firewall predicts and explains resistance that already exists. It never designs, modifies, or suggests changes to an organism.

MODULE 01

Genome Reader

A reconstructed FASTA is annotated with NCBI AMRFinderPlus — known resistance genes and point mutations mapped into a documented, repeatable feature space.

MODULE 02

Predictor

One calibrated model per antibiotic, trained only on laboratory-measured outcomes — de-duplicated by genetic similarity so it learns resistance, never look-alikes.

MODULE 03

Decision Report

Every drug gets a verdict, calibrated confidence, and an evidence category. Weak or unfamiliar evidence returns a no-call — a strength, not a failure.

Responsibility

Built to be trusted,
not just to be right.

NO DECISIONING

Never a treatment call

The report is decision support for professionals. Every result carries the requirement: confirm with standard laboratory testing.

HONEST UNCERTAINTY

Calibrated, with no-calls

Confidence scores are calibrated against reality. Weak, conflicting, or unseen-group evidence returns no-call instead of false confidence.

HONEST GENERALIZATION

Grouped splits

Near-identical genomes never cross train/test. Performance is reported by genetic group, including groups never seen in training.

HONEST EXPLANATIONS

Evidence classes

Known resistance markers are separated from statistical associations. A feature-importance score is not a biological cause.

DEFENSIVE BY DESIGN

No organism design

The system never generates, modifies, or optimizes organisms. It exists to slow resistance and protect patients.

CLEAN DATA

Lab-measured only

Training labels come from laboratory tests in BV-BRC — never computational phenotype predictions. No circular learning.
